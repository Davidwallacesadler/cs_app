# Practice 2.3

A single byte can be represented by 2 hexadecimal digits. Fill in the missing entries in the following table, giving the decimal, and hexadecimal values of different byte patterns:  

| row     | dec     | bin       | hex     |
| ------- | ------- | -------   | ------- |
| A.      | 0       | 0000 0000 | 0x00    |
| [B.](#b)| 158     | _         | _       |
| [C.](#c)| 76      | _         | _       |
| [D.](#d)| 145     | _         | _       |
| [E.](#e)| _       | 1010 1110 | _       |
| [F.](#f)| _       | 0011 1100 | _       |
| [G.](#g)| _       | 1111 0001 | _       |
| [H.](#h)| _       | _         | 0x75    |
| [I.](#i)| _       | _         | 0xBD    |
| [J.](#j)| _       | _         | 0xF5    |

---

## B.

Given `158 (dec)`

`hex` - We use the method outlined in the book dividing by 16 repeatedly to get the place values:
```
158 = 9 * 16 + 14 (9) (16^1)
14  = 0 * 16 + 14 (E) (16^0)
-----------------
=> 0x9E
```

`bin` - We can just convert the hex value to binary:
```
0x    9    E
   1001 1110
------------
=> 1001 1110
```

## E.

Given `1010 1110`:

`hex`:
```
   1010 1110
0x    A    E
------------
=> 0xAE
```

`dec` - Just use the base 10 representation for hex:
```
0xA = 10
0xE = 14 

0x9C
=> (10* 16^1) + (14 * 16^0)
=> (10 * 16) + (14 * 1)
=> 160 + 14
---------------------------
=> 174
```

## F.

Given `0011 1100`:

`hex`:
```
   0011 1100
0x    3    C
------------
=> 0x3C
```

`dec`:
```
0x9 = 3  (16^1)
0xC = 12 (16^0)

0x3C
=> (3 * 16^1) + (12 * 16^0)
=> (3 * 16) + (12 * 1)
=> 48 + 12
---------------------------
=> 60
```

## G.

Given `1111 0001`:

`hex`:
```
   1111 0001
0x    F    1
------------
=> 0xF1
```

`dec`:
```
0xF = 15 (16^1)
0x1 = 1  (16^0)

0x9C
=> (15 * 16^1) + (1 * 16^0)
=> (15 * 16) + (1 * 1)
=> 240 + 1
---------------------------
=> 241
```
