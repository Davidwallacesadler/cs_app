# Practice 2.39

How could we modify the expression for form B for the case where bit position `n` is the most significant bit?

---

Let's write out the forms here so it is clearer what is being discussed:

Generalizing the task of generating code for the expression `x * K` for some constant `K`, the compiler can express the binary representation of `K`. This represnetation will contain a run of `1`s from bit position `n` down to bit position `m` (`n <= m`). We can compute the effect if these bits on the product using form A and B:
- Form A: `(x<<n) + ( x<<(n-1) ) + ... + (x<<m)`
- Form B: `( x<<(n+1) ) - (x<<m)`

So we want to know how we need to change the expression for Form B if `n` is at the most significant bit.

We know the issue comes from this expression `( x<<(n+1) )` - if we were to do this the value would not fit in the word size.

What if we had the case of multiplying by 14 and have `w=4`?
- 14 = [1110]
- n = 3 (`n = w - 1`)
- `(x<<4)` = 0 given (`w = 4`)

Lets generalize:
```
Suppose: n = w - 1
Then, using Form B we have:
  (x << (n+1) ) - (x<<m)
= (x << (w - 1 + 1) ) - (x<<m) (given n = w - 1)
= (x<<w) - (x<<m)
= 0 - (x<<m) (since x<<w evaluates to 0)
= -(x<<m)
```

So if `n` is in the most significant bit position, Form B becomes: `-(x<<m)`
