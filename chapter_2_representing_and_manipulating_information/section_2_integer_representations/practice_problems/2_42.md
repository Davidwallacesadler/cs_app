# Practice 2.42

Write a function `div16` that returns the value `x/16` for integer argument x. Your function should not use division, modulus, multiplication, any conditionals, any comparision operators, or any loops. You may assume that the data type `int` is 32 bits long and uses a two's-complement representation, and that right shifts are performed arithmetically.

---

We know from the derivation on the earlier pages we if we have C variables x and k where:
- x is a two's complement value
- k is a unsigned value such that 0 <= k < w
Then the C expression `x>>k` yields the floor of `x/2^k`

In our situation we have division by 16 which means k = 5 in this case since 2^4 = 16. Given this value of k which is greater than 0 and less than the word size (w = 32), we should be able to have a simple function that just right shifts the number by 4:

```c
int div16(int x) {
    return x >> 4;
}
```

Looking at the solution in the back it seems I have missed an idea around bias. Not really sure what they are talking about with this but the mention we need to compute the bias using a trick where we shift x to the right by 31 (w-1) which would generate a word with all 1s if x is negative (arithmetic right shift) and all 0s otherwise. Then we mask the appropriate 4 bits using a bit mask. Not really sure the reasoning behind this but just noting it here.

```c
int div16(int x) {
    /* Compute bias to be either 0 (x >= 0) or 15 (x < 0) */
    int bias = (x >> 31) & 0xF;
    return (x + bias) >> 4;
}
```
