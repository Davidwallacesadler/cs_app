# Practice 2.29

Fill in the following table in the style of Figure 2.25. Give the integer values of the 5-bit arguments, the values of both their integer and two's-compliment sums, the bit-level representation of the two's-complement sum, and the case from the derivation of Equation 2.13

| row     | x           | y          | x + y | x +(t/5) y | case |
| ------- |  ---------- | ---------- | ----- | ---------- | ---- |
| [A.](#a)| [10100]     | [10001]    | ___   |  ___       |  ___ |
| [B.](#b)| [11000]     | [11000]    | ___   |  ___       |  ___ |
| [C.](#c)| [10111]     | [01000]    | ___   |  ___       |  ___ |
| [D.](#d)| [00010]     | [00101]    | ___   |  ___       |  ___ |
| [E.](#e)| [01100]     | [00100]    | ___   |  ___       |  ___ |

---

For each row in the table:
1. Interpret x and y binary values as two's complement integers (x) and (y).
2. Take x and y and add them bitwise while not thinking about word size (x + y).
3. Then truncate the sum to the 5-bit word size (x +(t/5) y).
4. Lastly, we will interpret if the value positve overflowed, negative overflowed, or was a normal operation (case).

## A.
```
(x):
    0b10100 = -(1 * 2^4) + (1 * 2^2) = -16 + 4 = -12

(y):
    0b10001 = -(1 * 2^4) + (1 * 2^0) = -16 + 1 = -15

(x + y):
   1 
    10100
    10001 +
    -------
   100101 = -(1 * 2^5) + (1 * 2^2) + (1 * 2^0) = -32 + 4 + 1 = -27

(x +(t/5) y):
    100101
    011111 &
    --------
     00101 = (1 * 2^2) + (1 * 2^0) = 4 + 1 = 5

(case):
x = -12  => x < 0
y = -15  => y < 0
s = 5    => s > 0
         --------
         Negative Overflow (1)
```
