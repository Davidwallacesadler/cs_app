# Practice 2.36

For the case where data type `int` has 32 bits, devise a version of `tmult_ok` (Problem 2.35) that uses the 64-bit precision of data type `int64_t`, without using division.

---

Thinking about doing this similar to how the book discussed doing this by hand where we will perform the multiplication, then truncate to the word size and check the resulting value.

```c
int tmult_ok(int x, int y) {
    // If x or y is zero we know we are good
    if (x == 0 || y == 0)
        return 1

    // Since int has 32 bits, we should be able to express the full product in 64 bits
    int64_t product = x * y;

    // How do we truncate the value to 32 bits?
    // We could use a bit mask to pull out these values!
    // We want a mask that has 1s up to 32 bits and 0s everywhere else
    int truncated = product & 0xFFFFFFFF

    // Just going to break down all the cases that make sense to me:
    // 1. x < 0, y < 0, p < 0 -- (both negative, product negative)
    // 2. x > 0, y > 0, p < 0 -- (both positive, product negative)
    // 3. x > 0, y < 0, p > 0 -- (one positive - one negative, product positive)
    // 4. x < 0, y > 0, p > 0 -- (one positive - one negative, product positive)
    if ((x < 0 && y < 0 && p < 0) || (x > 0 && y > 0 && p < 0) || (x > 0 && y < 0 && p > 0) || (x < 0 && y > 0 && p > 0))
        return 0

    // If we get here we are good
    return 1
}
```

Looking at the solution in the back makes a bit more sense haha. If the product in 64 bits is the same as the 32 bit representation we know we did not overflow! We can use casting to facilitate this!

```c
int tmult_ok(int) {
    int64_t product = x * y;        // Compute the full-precision product
    return product == (int) product // See if the full-precision product is the same as the truncated one
}
```
