# Practice 2.18

In Chapter 3, we will look at listings generated by a _disassembler_, a program that converts an executable program file back to a more readable ASCII form. These files. contain many hexadecimal numbers, typically representing values in two's complement form. Being able to recognize these numbers and understand their signifigance (for example, whether they are negative or positive) is an important skill.

For the lines labled A-I (on the right) in the following listing, convert the hexadecimal values (in 32-bit two's complement for) shown to the right of the instruction names (`sub`, `mov`, and `add`) into their decimal equivalents:


4004d0:  48 81 ec e0 02 00 00  sub  $0x2e0,%rsp             [A.](#a)

4004d7:  48 8b 44 24 a8        mov  -0x58(%rsp),%rax        [B.](#b)

4004dc:  48 03 47 28           add  0x28(%rdi),%rax         [C.](#c)

4004e0:  48 89 44 24 d0        mov  %rax,-0x30(%rsp)        [D.](#d)

4004e5:  48 8b 44 24 78        mov  0x78(%rsp),%rax         [E.](#e)

4004ea:  48 89 87 88 00 00 00  mov  %rax,0x88(%rdi)         [F.](#f)

4004f1:  48 8b 84 24 f8 01 00  mov  0x1f8(%rsp),%rax        [G.](#g)

4004f8:  00

4004f9:  48 03 44 24 08        add  0x8(%rsp),%rax          

4004fe:  48 89 24 c0 00 00     mov  %rax,0xc0(%rsp)         [H.](#h)

400505:  00

400506:  48 8b 44 d4 b8        mov  -0x48(%rsp,%rdx,8),%rax [I.](#i)

## A.

`0x2e0` to bin:

```
0x   2    e    0
  0010 1110 0000
----------------
= 0010 1110 0000
```

B2T (bin to two's complement):

```
0010 1110 0000
= (1 * 2^9) + (1 * 2^7) + (1 * 2^6) + (1 * 2^5)
= 512 + 128 + 64 + 32
---------------------
= 736
```

## B.

`-0x58` to bin:

```
- 0x    5    8
-    0101 1000
--------------
= - 0101 1000
```

B2T:

```
- 0101 1000
= -1 * ((1 * 2^6) + (1 * 2^4) + (1 * 2^3))
= -1 * (64 + 16 + 8)
= -88 (0xA8)
```

## C.

`0x28` to bin:

```
0x   2    8
  0010 1000
-----------
= 0010 1000
```

B2T:

```
0010 1000
= (1 * 2^5) + (1 * 2^3)
= 32 + 8
= 40
```

## D.

`-0x30` to bin:

```
- 0x   3    0
-   0011 0000
-----------
= - 0011 0000
```

B2T:

```
- 0011 0000
= -1 * ((1 * 2^5) + (1 * 2^4))
= -1 * (32 + 16)
= -48 (since value is negative)
```

## E.

`0x78` to bin:

```
0x   7    8
  0111 1000
-----------
= 0111 1000
```

B2T:

```
0111 1000
= (1 * 2^6) + (1 * 2^5) + (1 * 2^4) + (1 * 2^3)
= 64 + 32 + 16 + 8
= 120
```

## F.

`0x88` to bin:

```
0x   8    8
  1000 1000
-----------
= 1000 1000
```

B2T:

```
1000 1000
= (1 * 2^7) + (1 * 2^3)
= 128 + 8
= 136
```

## G.

`0x1f8` to bin:

```
0x   1    f    8
  0001 1111 1000
----------------
= 0001 1111 1000
```

B2T:

```
0001 1111 1000
= (1 * 2^8) + (1 * 2^7) + (1 * 2^6) + (1 * 2^5) + (1 * 2^4) + (1 * 2^3)
= 504
```

## H.

`0xc0` to bin:

```
0x   c    0
  1010 0000
-----------
= 1010 0000
```

B2T:

```
1010 0000
= (1 * 2^7) + (1 * 2^5)
= 160
```

## I.

`-0x48` to bin:

```
- 0x   4    8
-   0100 1000
-----------
= - 0100 1000
```

B2T:

```
- 0100 1000
= -1 * ((1 * 2^6) + (1 * 2^3))
= -72
```
