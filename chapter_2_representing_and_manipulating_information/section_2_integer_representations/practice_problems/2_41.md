# Practice 2.41

For a run of ones starting at bit position `n` down to bit position `m` (`n >= m`), we saw that we can generate two forms of code, A and B. How should the compiler decide which form to use?

---

So in this scenario, we have some number that has a run of 1's starting at position n down to position m such that n is at or greater than m.

So something like this:
```
[1111 1111 ... 1100]
 ^               ^
 n               m
```

We talked about generating two different forms:
- Form A: Representing the product using successive addition with bit shifts starting at n and working down to m.
```
(x<<n) + (x<<(n - 1)) + ... + (x<<m)
```
- Form B: Represneting the product as shifts startgin at `n + 1` and subracting up to m.
```
(x<<(n + 1)) - (x<<m)
```

The compiler should choose the form that is the most optimal meaning it should choose the form with the smallest number of shifts, adds, and subractions.

How should the compiler decide this? Lets break down the `n >= m` range into three:
1. Where `n = m`
2. Where `n = m + 1`
3. Where `n > m + 1`

If `n = m`: Form A only requires a single shift while Form B requires 2 shifts and a subtraction. So Form A should be used in this case.
```
(x<<n) + (x<<(n - 1)) + ... + (x<<m)
=> (x<<n) when n = m
```

If `n = m + 1`: In this cases the Forms would be equivalent as they would both require two shifts and a arithmetic operation. So EITHER Form A or Form B would work just fine here.

If `n > m + 1`: Form B only requires 2 shifts and a subtraction in comparrison to Form A which could have more shifts and more additions (just by the definition of the form)

So the compiler should choose:
- If `n = m` => Form A
- If `n = m + 1` => Form A OR Form B (assuming addition and subtraction are roughly equivalent work)
- If `n > m + 1` => Form B
